# Security Policy

This repository contains documentation, lab guidance, and publishing
automation — not a deployed service. Even so, please report:

- Any committed credential, key, token, or personal data found in the
  repository history.
- Any lab or configuration example that recommends an insecure default
  (weak ciphers, disabled authentication, overly broad firewall rules)
  without a clear "for lab use only" warning.
- Vulnerabilities in the publishing/build scripts under `scripts/`.

## Reporting

Open a private security advisory on the repository, or contact the
repository owner directly. Do not open a public issue for a credential leak.

## Scope

Vendor product vulnerabilities (VMware, Cisco, Palo Alto Networks, Fortinet,
Dell, Forescout, Gigamon, AWS, Red Hat, Canonical, etc.) are out of scope for
this repository — report those to the vendor directly.
